## Core Concepts

### `RecommendationStatus`

Defines the current state of a pricing recommendation.

#### `active`
Recommendation is current and actionable. Displayed prominently to user.

#### `dismissed`
User has chosen to ignore this recommendation. Hidden from main view but preserved in history.

#### `implemented`
User has marked recommendation as completed. Moved to implementation history.

#### `expired`
Recommendation is no longer relevant due to data changes or time passing. Automatically hidden.

### `RecommendationPriority`

Indicates the urgency and potential impact of a recommendation.

#### `high`
Significant revenue impact potential (>10% MRR increase) or urgent competitive action needed. Displayed at top of list with visual emphasis.

#### `medium`
Moderate impact potential (3-10% MRR increase) or important optimization opportunity.

#### `low`
Minor optimization or informational insight. Lower priority in display.

### `RecommendationType`

Categories of recommendations generated by the system.

#### `priceGap`
User's pricing is significantly different from competitor average. Suggests price adjustment.

#### `planAdoption`
Uneven distribution of customers across pricing tiers. Suggests tier restructuring or feature limits.

#### `missingTier`
Analysis suggests adding new pricing tier (e.g., Enterprise) based on customer clustering.

#### `competitorChange`
Competitor recently changed pricing. Suggests reviewing own pricing in response.

#### `revenueOptimization`
Opportunity to increase revenue through pricing adjustments based on customer behavior.

## Workflow

### Recommendation Generation: `—` → `active`
- System analyzes billing data and competitor prices daily
- Rule engine evaluates conditions for each `RecommendationType`
- New recommendation created when conditions met
- → Assigned `RecommendationPriority` based on impact
- Occurs automatically at 4:00 AM UTC after daily data sync

### `active` → `dismissed`
- User clicks "Dismiss" on recommendation
- → Moved to dismissed history
- → Hidden from active recommendations list
- Triggered by authenticated user

### `active` → `implemented`
- User marks recommendation as "Implemented"
- → Moved to implementation history
- → Used for tracking recommendation effectiveness
- Triggered by authenticated user

### `active` → `expired`
- Underlying data changes make recommendation obsolete
- Recommendation age exceeds 90 days without action
- → Automatically hidden from active list
- → Preserved in history for analysis

### `dismissed` → `active`
- User can restore dismissed recommendation
- Recommendation reappears in active list
- Triggered by authenticated user from dismissed history

## Discovery & Display

### Active Recommendations Dashboard

Main view showing current actionable recommendations.

#### Filters
- Priority: `high`, `medium`, `low`
- Type: `priceGap`, `planAdoption`, `missingTier`, `competitorChange`, `revenueOptimization`

#### Sorting
- Primary: `RecommendationPriority` (high to low)
- Secondary: Potential impact (highest to lowest)
- Tertiary: Date created (newest first)

#### Displayed Data
- Recommendation title and description
- `RecommendationPriority` with visual indicator
- `RecommendationType` category
- Estimated revenue impact (if calculable)
- Specific action suggested
- Supporting data and rationale
- Date generated

#### Default State
- Shows all recommendations with `RecommendationStatus` of `active`
- Sorted by priority (high first)

### Recommendation Detail View

Detailed analysis for a single recommendation.

#### Displayed Data
- Full recommendation description
- `RecommendationType` and `RecommendationPriority`
- Detailed rationale with data points
- Competitor comparison data (if applicable)
- Current vs. recommended pricing
- Estimated impact calculation breakdown
- Historical context
- Action steps to implement

### Recommendation History

Archive of past recommendations and their outcomes.

#### Filters
- Status: `dismissed`, `implemented`, `expired`
- Date range: Last 30 days, Last 90 days, All time

#### Displayed Data
- Recommendation title
- `RecommendationStatus` and date of status change
- Original `RecommendationPriority`
- Whether user took action
- Historical data snapshot

## Business Rules

### Generation Rules
- Recommendations only generated when `StripeConnectionStatus` is `connected`
- Requires minimum 30 days of billing data for accurate analysis
- Requires at least 1 competitor with `CompetitorStatus` of `active` for comparison recommendations
- Maximum 10 active recommendations shown at once (prioritized by impact)

### Rule Definitions by Type

#### `priceGap` Rules
- Triggered when user's price is >30% below competitor average
- Triggered when user's price is >50% above competitor average
- Calculates potential MRR impact based on current customer count
- Suggests specific price point based on competitor median

#### `planAdoption` Rules
- Triggered when >60% of customers on lowest tier
- Triggered when >50% of customers at highest tier limit
- Suggests adding feature limits or tier restructuring
- Calculates potential upgrade revenue

#### `missingTier` Rules
- Triggered when >50% of highest tier customers at usage limits
- Triggered when significant gap between tier prices (>3x difference)
- Suggests Enterprise tier pricing based on current highest tier + 50-100%
- Estimates potential revenue from new tier

#### `competitorChange` Rules
- Triggered when competitor increases prices by >10%
- Triggered when competitor adds new tier
- Triggered when multiple competitors make similar changes within 30 days
- Suggests reviewing own pricing in context of market movement

#### `revenueOptimization` Rules
- Triggered when ARPU is >20% below industry benchmark
- Triggered when churn rate increases after price change
- Suggests A/B testing price points
- Calculates expected value of optimization

### Priority Assignment
- `high`: Potential impact >$1,000 MRR or >10% revenue increase
- `medium`: Potential impact $300-$1,000 MRR or 3-10% revenue increase
- `low`: Potential impact <$300 MRR or <3% revenue increase

### Expiration Rules
- Recommendations expire after 90 days if not actioned
- Recommendations expire immediately if underlying data changes significantly
- `competitorChange` recommendations expire after 30 days

### Data Freshness
- Recommendations regenerated daily based on latest data
- All changes to recommendation status are immediately visible
- Impact calculations update when billing data syncs

### Validation Rules
- Cannot dismiss recommendation that is already `dismissed` or `implemented`
- Cannot mark as `implemented` without user confirmation
- Restored recommendations maintain original creation date but updated impact calculations
