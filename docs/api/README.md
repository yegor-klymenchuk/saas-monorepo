# API Documentation

This folder contains technical API documentation for the CA application backend endpoints.

## What belongs here?

- **HTTP endpoint specifications** (request/response contracts)
- **API conventions and standards** (authentication, error handling, pagination)
- **Technical implementation details** for backend services
- **Data transfer object (DTO) schemas** and validation rules
- **Not:** Business logic, user workflows, or feature descriptions (see [Features](../features/README.md))

## Core API Conventions

Before diving into specific endpoints, familiarize yourself with these foundational specifications that apply across all APIs:

- [Conventions](./conventions.md) - Common standards for ID formats, date/time handling, error responses, and null values (applies to both HTTP and WebSocket)
